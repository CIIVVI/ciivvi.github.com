By Patrick Cheng

Published 2015-06-05

Contents

LeetCode
Clarifying questions:

  • min() behavior: empty stack?
    Analysis
  • difference to regular stack: min() in O(1), CQ on min().

  • idea: record current min

    • if push, update min value
    • if pop, restore previous value of min
      • $\Rightarrow$ need extra structure to store minvalue for $\color{red}{\text{every state}}$ of stack
      • $\Rightarrow$ use same structure, a stack, to store the min value of current state.

  • time: O(1), space: O(N)

Improvement

  • minSt pushs when x <= minSt.top()
  • minSt pops when datSt.top() == minSt.top();
