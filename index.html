Max Tree

By Patrick Cheng

Published 2015-06-05

LintCode
Analysis

  • find max element as root, recursively build LST and RST $\Rightarrow$ $O(N^2)$ in worst case.
  • start with example: [2, 5, 6, 0, 3, 1]
  • observation:
    • select 6 as root, left child is the largest element to its left, right child is the largest element to its right.
    • view 5, its parent is the first greater element to its right.
    • view 3, its parent is the first greater element to its left.
    • for a given element, its parent is the smaller between: first greater element to the left / right.
  • idea: bottom-up approach
  • another idea:
    • if curVal < preVal, preNode->right = curNode
    • if curVal > preVal, trace back till curVal < nodeVal, set node->right = curNode, RSL of node be LST of curNode.
    • need to maintain a stack of path to root.

Time: O(N), space: O(N) worst case

Improvement

  • remove if / else redundancy
  • adopt vector instead of stack

get index of first smaller element to the left

By Patrick Cheng

Published 2015-06-05

Question:

For every element in an array, find the index of first smaller value to the left. If not found, set to -1.
Example: a = [2 1 3 4], return [-1 -1 1 2].
Challenge: O(N) time.

Analysis:

  • exhaustive search: O(N^2)
  • start with example a = [1,5,4,2]
    • for 1, search []
    • for 5, search [1]
    • for 4, search [1,5]
    • for 2, search [1,5,4]
      It worths noting that comparing to 5 is not necessary, since searching is from right to left, target has already compared to 4.
  • new strategy:
    • for 1, search []
    • for 5, search [1]
    • for 4, search [1,5] and delete 5
    • for 2, search [1,4] and delete 4
  • store the searching range with a stack. Every element was pushed and popped once. Time: O(N).

Time: O(N).

Clarifying questions:

  • 00001 $\Rightarrow$ 00010
  • 00010 $\Rightarrow$ 00100
  • 00011 $\Rightarrow$ 00101
  • 00101 $\Rightarrow$ 00110
  • 10110 $\Rightarrow$ 11001
  • 0? all 1? all 1 with trailing 0’s?
  • $\color{red}{\text{Type of input, int? unsigned int?}}$

Analysis:
10110 $\Rightarrow$ 11010 $\Rightarrow$ 11001

  • swapping a 1-bit with a 0-bit to its left increase the number.
  • swapping a 1-bit with a 0-bit to its right decrease the number.
  • given k 1’s, the smallest number is 001..1.

Algorithm:

  • increase first, then decrease to minimal.
  • find the rightmost non-trailing 0-bit (b-th bit from right), swap with the bit to its right.
  • flush all the 1’s under b-th bit to the right. $\Rightarrow$ count #1-bit and set.
